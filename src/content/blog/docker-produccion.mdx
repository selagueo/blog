Después de correr containers en producción por 2 años, aprendí varias cosas por las malas.

## Errores que cometí

1. **Imágenes de 2GB** — no usaba multi-stage builds
2. **Root por defecto** — siempre crear un usuario non-root
3. **No limitar recursos** — CPU y memoria sin límites = desastre

## Buenas prácticas

- Usá Alpine o distroless como base
- Cada contenedor hace una sola cosa
- Health checks siempre

## Herramientas que recomiendo

Docker Compose para dev, Kubernetes para producción (si lo necesitás).
